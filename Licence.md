## Use Cases and Permissions for Each License

| License                        | Typical Use Cases                                                                 | What Is Allowed                                                                                   | What Is Not Allowed                                              |
|---------------------------------|----------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------|------------------------------------------------------------------|
| **GNU GPL**                     | Open source projects needing strong copyleft (e.g., Linux, WordPress)            | - Use, modify, distribute code<br>- Create derivatives (must be GPL)<br>- Commercial use          | - Combine with proprietary code<br>- Release derivatives under a non-GPL license |
| **Apache License 2.0**          | Open source libraries, enterprise software (e.g., Hadoop, Kubernetes)            | - Use, modify, distribute<br>- Commercial and proprietary use<br>- Patent rights granted          | - Remove license/NOTICE files<br>- Use trademarks without permission |
| **MIT License**                 | Small projects, web tools, libraries (e.g., jQuery, Rails)                       | - Use, modify, distribute<br>- Commercial and proprietary use                                     | - Remove copyright/license notice                                |
| **BSD License (2-Clause, 3-Clause)** | Academic, research, and commercial software (e.g., FreeBSD, SQLite)             | - Use, modify, distribute<br>- Commercial and proprietary use                                     | - Use author’s name for endorsement<br>- Remove copyright/license notice |
| **Mozilla Public License (MPL)**| Projects mixing open and closed source (e.g., Firefox)                           | - Use, modify, distribute<br>- Combine with proprietary code<br>- Only modified MPL files open     | - Release modified MPL files as closed source                    |
| **Creative Commons (CC-BY, CC-BY-SA)** | Documentation, media, educational content (not software code)                   | - Use, share, adapt<br>- Commercial use (with attribution)<br>- Derivatives (SA: same license)    | - Use for software code<br>- Omit attribution<br>- Release derivatives under a different license (SA) |

---

**Summary:**  
Choose a license based on your project’s needs for openness, commercial use, and compatibility. Always comply with attribution, redistribution, and copyleft requirements to avoid legal issues.
