## Step-by-Step Markdown Examples (Expanded)

Below are detailed, step-by-step examples of common Markdown syntax. Each section includes the Markdown input, a description, and the resulting output.

### 1. Headings

Headings organize content into sections. Use `#` for different heading levels.

**Markdown Input:**
```markdown
# Heading 1
## Heading 2
### Heading 3
```

**Description:**  
- `#` creates a top-level heading.  
- `##` creates a second-level heading.  
- `###` creates a third-level heading.

**Result:**

# Heading 1
## Heading 2
### Heading 3

---

### 2. Bold and Italic

Emphasize text using asterisks or underscores.

**Markdown Input:**
```markdown
**Bold Text**
*Italic Text*
```

**Description:**  
- `**Bold Text**` or `__Bold Text__` makes text bold.  
- `*Italic Text*` or `_Italic Text_` makes text italic.

**Result:**

**Bold Text**  
*Italic Text*

---

### 3. Lists

Create ordered and unordered lists for organization.

**Markdown Input:**
```markdown
- Item 1
- Item 2

1. First
2. Second
```

**Description:**  
- Use `-`, `*`, or `+` for unordered lists.  
- Use numbers followed by a period for ordered lists.

**Result:**

- Item 1
- Item 2

1. First
2. Second

---

### 4. Links and Images

Add hyperlinks and display images.

**Markdown Input:**
```markdown
[GitHub](https://github.com)
![Logo](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)
```

**Description:**  
- `[Text](URL)` creates a clickable link.  
- `![Alt Text](Image URL)` displays an image.

**Result:**

[GitHub](https://github.com)  
![Logo](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)

---

### 5. Code

Show code snippets inline or in blocks.

**Markdown Input:**
```markdown
Inline `code`
```
Code block
```
```

**Description:**  
- Use backticks `` ` `` for inline code.  
- Use triple backticks for code blocks.

**Result:**

Inline `code`

```
Code block
```

---

### 6. Blockquotes

Highlight quoted text.

**Markdown Input:**
```markdown
> This is a quote.
```

**Description:**  
- Use `>` at the start of a line for blockquotes.

**Result:**

> This is a quote.

---

### 7. Tables

Organize data in rows and columns.

**Markdown Input:**
```markdown
| Name  | Role   |
|-------|--------|
| Alice | Admin  |
| Bob   | User   |
```

**Description:**  
- Use pipes `|` and dashes `-` to create tables.

**Result:**

| Name  | Role   |
|-------|--------|
| Alice | Admin  |
| Bob   | User   |

---

Each example includes the Markdown syntax, a brief explanation, and the rendered output to help you understand Markdown step by step.
